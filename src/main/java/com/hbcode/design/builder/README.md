# 建造者模式

### 一 定义
1.  使用多个简单的对象一步一步构建成一个复杂的对象  
2.  指将一个复杂对象的构造与对象本身分离 **(将对象和对象的创建过程分开)** 这样隔离后我们可以在同样的构建过程可以创建不同的对象表示.
### 二 理解
1.  关键是将对象和对象的创建隔离,使得同样的构建过程可以创建不同的表示 
### 三 角色
1. 产品角色 Product 实际生成的产品（对象）
2. 抽象建造者 Builder 定义了一个产品（对象）的创建需要哪些过程
3. 具体建造者 ConcreteBuilder 实现了 抽象建造者 Builder接口
4. 指挥者 Director  使用 Builder 建造者创建产品（对象）。它主要是用于创建一个复杂的对象。它主要有两个作用，一是:隔离了客户与对象的生产过程，二是:负责控制产品对象的生产过程。
